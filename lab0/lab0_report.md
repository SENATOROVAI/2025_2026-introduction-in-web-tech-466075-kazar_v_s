# Отчёт по лабораторной работе №0: Создание репозитория и настройка рабочего окружения


## Цель работы
Научиться создавать репозитории, настраивать рабочее окружение и изучать основы работы с Git и GitHub.


## Ход работы
1) Лабораторная работа началась с подготовки аккаунта и окружения. Сначала, поскольку аккаунт на GitHub уже существовал, я перешёл к настройке SSH-ключей, чтобы обеспечить безопасный и удобный доступ к репозиториям без ввода пароля каждый раз. Для этого в терминале (я использовал Git Bash на Windows) я сгенерировал пару ключей командой `ssh-keygen -t ed25519 -C "kazar.vladimir@gmail.com"`. Я нажал Enter три раза, чтобы создать ключи без пароля (для учебных целей), и ключи сохранились в папке `C:\Users\Имя_пользователя\.ssh\`. Затем я скопировал публичный ключ и добавил его в настройки GitHub: зашёл в профиль, Settings, SSH and GPG keys, New SSH key, вставил ключ и сохранил. Для проверки подключения выполнил `ssh -T git@github.com`, и после подтверждения отпечатка сервера (введя "yes" на запрос) увидел сообщение о успешной аутентификации.

2) Далее я создал новый репозиторий на GitHub. Зашёл на https://github.com/new, указал имя `devops-lab-kazar`, сделал его публичным без инициализации README или других файлов, и создал. Это обеспечило пустой репозиторий, готовый к клонированию. Затем склонировал его на локальный компьютер: открыл терминал, перешёл в удобную директорию и выполнил `git clone`. После этого зашёл в папку репозитория командой `cd devops-lab-kazar`. Теперь репозиторий был локально доступен, и я мог работать с файлами.

3) Следующим шагом стало создание файла `README.md`. Я открыл папку в VS Code (командой `code .` в терминале) и создал файл `README.md`. В него я добавил заголовок проекта, описание (что это репозиторий для изучения DevOps в рамках курса), контактные данные (ФИО, группа, email, GitHub), и план изучения: список чекбоксов с темами от Git до мониторинга. Это сделано для того, чтобы репозиторий был информативным и удобным для просмотра. 

4) Затем я создал файл .gitignore, чтобы исключить ненужные файлы из коммитов, такие как системный мусор от Windows. В VS Code создал файл с именем .gitignore и добавил в него стандартные исключения для Windows (Thumbs.db, Desktop.ini и т.д.).

5) После этого я создал ветку `develop` для дальнейшей работы, чтобы не изменять main напрямую. Выполнил команду `git checkout -b develop`, и терминал подтвердил переключение на новую ветку. 

6) В ветке develop я создал файл `CONTRIBUTING.md`, где описал правила участия в проекте: как создавать ветки, коммитить, открывать Pull Request'ы. Это полезно для потенциальных соавторов. 

7) Теперь, когда все файлы были готовы, я добавил их в индекс Git командой `git add .` (чтобы добавить все изменения сразу), затем сделал коммит с сообщением `git commit -m "Initial project setup"`. Это зафиксировало начальную настройку проекта. Далее запушил ветку на GitHub: `git push origin develop`. GitHub создал ветку удалённо.

8) На сайте GitHub я открыл репозиторий `devops-lab-kazar`, увидел уведомление о новой ветке и нажал `"Compare & pull request"`. Указал base как `main`, compare как `develop`, добавил заголовок "Initial project setup" и описание: "Добавлены README.md, .gitignore, CONTRIBUTING.md; начальная настройка проекта". Создал Pull Request. Затем на странице PR нажал "Merge pull request", подтвердил, и изменения из `develop` попали в `main`.

9) Наконец, удалил ветку `develop` кнопкой "Delete branch" на той же странице, чтобы репозиторий оставался чистым.


## Команды

- Генерация SSH-ключа:
```
ssh-keygen -t ed25519 -C "kazar.vladimir@gmail.com"
clap <c\Users\...\.ssh\id_ed25519.pub  # для копирования публичного ключа
ssh -T git@github.com  # проверка подключения
```

- Клонирование репозитория:
```
cd D:\Programming\27_02_2026_introduction-in-web-tech\lab0
git clone git@github.com:мой_логин/devops-lab-kazar.git
cd devops-lab-kazar
```

- Создание и добавление файлов в main (до ветки):
```
git add README.md .gitignore
git commit -m "Add README.md and .gitignore"
git push origin main
```

- Работа в ветке develop:
```
git checkout -b develop
# Создание CONTRIBUTING.md в редакторе
git add CONTRIBUTING.md
git commit -m "Initial project setup"
git push origin develop
```
